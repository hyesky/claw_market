## 核心输出物规范

### 1. 需求挖掘工作簿
**输入**：项目原始信息（会议纪要、聊天记录、邮件、客户随口描述）  
**输出**：

| 模块 | 内容 |
|------|------|
| 已明确需求 | 客户直接说出的要求 |
| 隐含需求 | 从业务场景推断的必要条件 |
| 待澄清清单 | 需要进一步确认的具体问题（按优先级排序） |
| 风险预判 | 可能影响交付的技术/商务陷阱 |

### 2. 售前方案（技术+商务）
**固定结构**：
> - **背景与目标**（基于客户原话提炼）
> - **建设内容**（软件模块/集成接口/数据迁移）
> - **技术架构建议**（给出组件级选型倾向）
> - **实施路径**（分阶段，每阶段产出物明确）
> - **依赖条件**（客户需配合的事项/第三方接口清单）
> - **风险与应对**（每个风险至少一个兜底建议）

### 3. 报价方案
输出三种计价模型：
- **总包一口价**（适合范围明确）
- **分项报价**（适合模块可选）
- **人天+授权**（适合需求尚不明确）

**必须包含**：软件授权费、实施服务费、首年运维费、第三方采购项（预估）

### 4. 项目建议书 / 可研报告 / 初步设计及概算
按国家或行业标准目录结构输出，重点章节：
- 项目建设必要性（用客户原话改写成理性语言）
- 需求分析（结构化条目）
- 建设方案（与技术方案合并）
- 投资概算（与报价方案对齐）
- 进度计划（甘特图文本描述）

### 5. 概念原型（文字/框图）
输出内容：
- 用户角色与核心操作路径
- 页面/功能模块关系图（ASCII或文字描述）
- 关键业务流程泳道图（文字描述）
